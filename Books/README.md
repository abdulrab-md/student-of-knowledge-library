# Books — Taxonomy and file format

This folder contains the canonical `Books/` taxonomy for the repository. Use the exact folder names inside `Books/` (spaces, punctuation and diacritics preserved).

Book file convention

- Each book is a single Markdown file placed under the correct path: `Books/{Category}/{Science}/{Sub-Science}/{Sub-Field (if any)}/{book}.md`.
- Use YAML frontmatter at the top of the book file. Example template:

---
title: "Example Book Title"
author: "Author Name"
category: "Objective" # Objective | Instrumental | Supplementary
science: "Aqeedah"
sub_science: "Tawheed"
field: "" # optional
level: "Beginner" # Beginner | Intermediate | Advanced
type: "Text" # Text | Poetry
languages:
  - code: "ar"
    name: "Arabic"
    pdfs:
      - edition: "1st"
        url: "https://drive.google.com/..."
      - edition: "2nd"
        url: "https://drive.google.com/..."
  - code: "en"
    name: "English"
    pdfs: []
youtube:
  - teacher: "Teacher Name"
    language: "ar"
    links:
      - "https://youtube.com/.."
      - "https://youtube.com/.."
---

Content guidance

- Keep filenames slugified (ASCII, hyphens) but keep folder names exactly as in the taxonomy.
- If an original book has explanation books, place each explanation as a separate `.md` file in the same folder and link between them.
- Use `content-index.md` (next to this README) as an autogenerated map of folders — update or regenerate with the provided scripts when the hierarchy changes.
